# 每日工作报告生成器

每日工作报告生成器。汇总当天的 Git 代码提交、Claude Code 会话活动、工作进展，生成结构化 Markdown 日报。

## 触发关键词

以下任意表述均可触发此技能：

- 今日总结
- 总结一下今日的工作
- 总结一下工作
- 今天做了什么
- 工作报告
- 日报
- daily report
- 生成日报
- `/claude-daily-report`

**指定日期**: 在消息中附带日期可补生成指定日期的报告，例如 "总结一下 2026-06-16 的工作"。

## 数据源收集

按以下顺序并行收集数据，每个来源独立查询，单源失败不影响其他源。

### 1. Git 提交记录

搜索以下目录中的 Git 仓库：

```
常见目录: ~/projects, ~/work, ~/dev, ~/code, ~/Documents, 当前工作目录
```

对每个找到的仓库，查询当天（默认）或指定日期的提交：

```bash
git -C <repo-path> log --since="YYYY-MM-DD 00:00:00" --until="YYYY-MM-DD 23:59:59" \
  --format="%h %s (%an, %ar)" --no-merges
```

合并所有仓库的提交记录，按时间倒序排列。如果某个目录下没有 Git 仓库，跳过并记录在备注中。

### 2. Claude Code 会话摘要

读取 `~/.claude/projects/` 下的 JSONL 文件，筛选当天有活动的会话：

- 解析每条 JSONL 记录的 `timestamp` 字段
- 提取 `role` 为 `user` 的消息作为"讨论主题"
- 提取 `role` 为 `assistant` 的消息中涉及工具调用的类型（Edit/Write/Bash/Agent/Skill 等）作为"操作统计"
- 汇总成: 会话项目名、讨论主题列表、操作统计（文件编辑数、命令执行数、搜索次数、Agent 调用次数）

如果 JSONL 文件过大（超过 5000 行），仅读取尾部 2000 行并按时间戳过滤。

### 3. 工作日报模板

填充以下结构化模板：

```markdown
## ✅ 今日完成
- [从 Git 提交 + Claude 会话中自动提取的工作项]

## 📌 明日计划
- [待补充 — 用户可手动编辑]

## 🚧 阻塞项
- [无 / 待补充]

## 📝 备注
- [自动摘要: 今日活跃的 Git 仓库数、Claude 会话数、主要操作类型统计]
```

### 4. 自定义数据源（可选）

用户可以在触发时追加以下指令：

- `日报 --notes "自由文本"` : 在备注区追加用户的自由文本
- `总结工作 --since "2026-06-10" --until "2026-06-17"` : 自定义时间范围（周报模式）
- `总结 XX 项目的工作` : 仅汇总指定项目

## 输出

### 文件

写入 `~/daily-reports/YYYY-MM-DD.md`，当天重复生成则覆盖。

### 格式模板

```markdown
# 每日工作报告 — YYYY-MM-DD (星期X)

> 自动生成于 YYYY-MM-DD HH:MM | 数据来源: N 个 Git 仓库, N 个 Claude 会话
```

## 实现流程

1. **解析参数**: 确定目标日期（默认今天）、提取自定义备注、限定项目范围
2. **并行收集数据**:
   - Agent 1: 搜索常见目录找到 Git 仓库，查询当天提交记录
   - Agent 2: 解析 Claude 会话 JSONL 文件，提取当天活动摘要
3. **汇总生成**: 将收集到的数据填入模板
4. **写入文件**: 输出到 `~/daily-reports/YYYY-MM-DD.md`
5. **展示摘要**: 在终端打印关键指标概览，告知文件完整路径

## 文件

- 技能定义: `~/.claude/skills/claude-daily-report/SKILL.md`
- 日报输出: `~/daily-reports/YYYY-MM-DD.md`
